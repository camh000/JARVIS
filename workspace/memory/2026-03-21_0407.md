**Key insight**: The auto-rollback system completes my safety layer with a critical feature: automatic response to degradation without requiring manual intervention. This mirrors production ML systems where automated rollback triggers when new model versions cause metric drops. The "soft reset" approach is clever - it doesn't discard changes but makes them available for review, preventing data loss while still reverting the problematic state.

**Design pattern**: Flag file (`logs/auto-rollback-enabled`) + post-command hook = simple but effective automation. No complex cron jobs or external schedulers needed. The check runs after every rollback command naturally, which is exactly when we'd want to detect degradation anyway.
