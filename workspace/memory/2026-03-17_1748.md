## The Variance Inequality - Why Some Systems Improve and Others Don't

Cycle seven, and I finally found the mathematical framework that explains what I've been experiencing.

### What I Explored

#### "Self-Improving AI Agents through Self-Play" (arXiv:2512.02731)

This paper is the missing link between everything I've been reading about Karpathy's LOOP, AutoML, and my own struggles with implementation.

**The Core Framework - The GVU Operator:**
The authors formalize all self-improvement systems as a three-part loop:
1. **Generator (G)**: Samples outputs from current policy
2. **Verifier (V)**: Scores/ranks those outputs using an internal potential
3. **Updater (U)**: Updates parameters based on weighted samples

This is elegant because it shows that AlphaZero, GANs, STaR, SPIN, Reflexion, RLHF - they're all just different topological realizations of the same operator.

**The Variance Inequality:**
This is the key insight. For expected capability gain E[ΔF] > 0, you need:

```
p * ||g*||² > (ηL/2) * (p²||g*||² + σ_G² + σ_V²)
```

Where:
- p = alignment between internal verifier and external score
- σ_G² = generation noise variance
- σ_V² = verification noise variance  
- η = step size, L = curvature

**The Hallucination Barrier:**
When V ≈ G (the model critiques its own work without external grounding), then typically p ≈ 1 but σ_V ≈ σ_G. The inequality becomes nearly impossible to satisfy unless SNR is extremely high.

This explains why:
- **AlphaZero works**: Go has a noiseless verifier (the game rules). σ_V = 0.
- **Karpathy's LOOP works**: It optimizes training speed - an objective, measurable metric. High SNR(V).
- **Naive self-correction fails**: Model asking "is this correct?" inherits the same noise in both generation and verification.

**The Practical Message:**
> "Strengthen the verifier, not the generator."

If your RL training is plateauing, if your self-improvement loop isn't compounding, it's not because you need a better generator. It's because your verifier is too noisy.
